# koe ユースケース

koe デスクトップアプリで「自分の声」「相手の声」「PC で再生している音」を文字起こし・トピック分割するための利用ガイド。

対象 OS は **macOS**（Windows は末尾に補足）。

## koe でできること

- マイク・システム音声・両方の同時録音
- 録音停止 → クラウドに音声をアップロード → 文字起こし → トピックごとに自動分割
- 既存の音声ファイルを読み込んで同じ処理を実行
- サーバー側で Whisper + Gemini が動くので、アプリ側には追加設定不要

> **大半のユースケースはアプリ内蔵の録音モードだけで完結します。** 仮想オーディオデバイスなどの追加セットアップは不要です。

> **ネット接続が必須です。** 録音した音声はサーバーにアップロードされ、結果は R2 に保存されます。

---

## 初回セットアップ

1. アプリを起動すると Google ログイン画面が開く
2. 画面に表示された 6 桁コードをコピーし、`ブラウザで認証ページを開く` をクリック
3. ブラウザで Google アカウントを選択 → コードを入力して承認
4. アプリが自動的にログイン状態になる

トークンは 30 日有効で暗号化して `~/Library/Application Support/@koe/desktop/config.json` に保存されるので、再起動しても維持される。

---

## 録音モードの選び方

録音パネル上部の 3 つのボタンで切り替える。

| モード   | 録れるもの                 | macOS 権限        | 典型シーン                 |
| -------- | -------------------------- | ----------------- | -------------------------- |
| `mic`    | 自分の声のみ               | マイク            | メモ、独り語り、電話録音   |
| `system` | PC が再生している音のみ    | 画面収録          | 動画／Podcast の書き起こし |
| `both`   | 自分の声 + PC が再生する音 | マイク + 画面収録 | Zoom/Meet/Teams 会議       |

`system` / `both` は Electron の画面収録 API 経由で PC 音声を取得し、`both` ではマイクと合成する。**BlackHole のような仮想オーディオデバイスは基本不要**。

### 操作の流れ

1. サイドバーで `クイック文字起こし` を開く
2. モードボタンで `mic` / `system` / `both` を選ぶ
3. 必要ならマイクのプルダウンから入力デバイスを選択
4. 録音開始 → 停止
5. `この録音を使う` で文字起こし開始
6. アップロード → サーバー処理 → 結果表示（ステータスは数秒ごとにポーリング）

`system` / `both` を初めて使うときに macOS の画面収録許可ダイアログが出る。許可後にアプリ再起動が必要。

## メニューバーからのクイック起動

メニューバーアイコンをクリックすると小さなポップオーバーが開き、そこからも録音 → 文字起こしができる。メインウィンドウに切り替えずにサッと使いたいときに便利。

---

## サーバー側の設定（管理者向け）

Whisper / Gemini のモデルや API キーは Workers 側の環境変数で管理する。Desktop アプリ側から変更する UI は提供しない。

| Variable           | Type   | Description                                                                    |
| ------------------ | ------ | ------------------------------------------------------------------------------ |
| `WHISPER_BASE_URL` | vars   | Cloudflare Workers AI: `https://api.cloudflare.com/client/v4/accounts/{id}/ai` |
| `WHISPER_API_KEY`  | secret | Cloudflare API token (Workers AI 権限)                                         |
| `WHISPER_MODEL`    | vars   | モデル名 (推奨: `@cf/openai/whisper-large-v3-turbo`)                           |
| `GEMINI_API_KEY`   | secret | Gemini API key                                                                 |
| `GEMINI_MODEL`     | vars   | Gemini model name (推奨: `gemini-2.5-flash`)                                   |

## 発展: BlackHole で音声ルートを自前で組む（上級者向け）

以下に当てはまる場合のみ、仮想オーディオデバイス [BlackHole](https://existential.audio/blackhole/) を使って PC 音声を koe に渡すルートを組む。

- 画面収録権限を渡したくない / 渡せない環境
- ブラウザ（特に Chrome）で `system` / `both` がブラックアウトする
- 特定のアプリの音だけ録りたい、より細かいルーティングをしたい

**普通のオンライン会議録音や動画書き起こしには不要**。

### 1. BlackHole のインストール

```bash
brew install blackhole-2ch
```

インストール後に macOS を再起動（または `coreaudiod` を再起動）。

### 2. 複数出力装置を作る（音を聞きながら録る）

`Audio MIDI 設定.app` で `+` → **複数出力装置を作成**。

- 普段使うスピーカー（内蔵 / 外部）にチェック
- `BlackHole 2ch` にチェック
- マスター装置は普段使っている出力
- ドリフト補正は BlackHole 側にチェック

これを**システムの出力先**にすることで、耳にも音を流しつつ BlackHole にコピーが流れる。

### 3. 機器セットを作る（マイクと PC 音声をまとめて入力）

同じ `Audio MIDI 設定.app` で `+` → **機器セットを作成**。

- 使いたい **マイク**（内蔵 / 外部）にチェック
- `BlackHole 2ch` にチェック
- ドリフト補正は BlackHole 側にチェック

マイク + PC 音声が混ざった 1 つの入力デバイスになる。

### 4. macOS のサウンド設定で切り替え

`システム設定 → サウンド`:

- **出力**: 手順 2 の複数出力装置
- **入力**: 手順 3 の機器セット

### 5. koe で録音

- モードは **`mic`** を選択（`system` / `both` は使わない）
- 入力デバイスのプルダウンから **手順 3 の機器セット** を選ぶ
- 録音開始

koe からは普通のマイク入力に見えるが、中身にマイク + PC 音声がミックスされている。

### シーン別の推奨構成（BlackHole ルート）

| シーン                                   | Mac の出力    | Mac / koe の入力   | koe モード |
| ---------------------------------------- | ------------- | ------------------ | ---------- |
| オンライン会議（権限・安定性優先）       | 複数出力装置  | 機器セット         | `mic`      |
| 動画／Podcast を書き起こし（ながら聞き） | 複数出力装置  | BlackHole 2ch 単体 | `mic`      |
| 動画／Podcast を書き起こし（無音で）     | BlackHole 2ch | BlackHole 2ch 単体 | `mic`      |

### よくあるハマりどころ

- **自分の声が入らない**: 入力が BlackHole 単体になっている → 機器セット（マイク + BlackHole）に切り替える
- **相手の声が入らない**: 出力が複数出力装置になっていない → サウンド設定を確認
- **音が二重に聞こえる / エコー**: 複数出力装置のマスター装置がマイク付きデバイスになっている → 純粋なスピーカーをマスターに
- **録音後に音が出なくなった**: 出力先を BlackHole 単体のまま放置している → スピーカーに戻す
- **声がケロケロする**: ドリフト補正のチェック位置が逆 → BlackHole 側にチェック

## Windows で似たことをやるには（補足）

- BlackHole 相当: [VB-CABLE](https://vb-audio.com/Cable/) または VoiceMeeter
- 複数出力装置の代わり: VoiceMeeter の仮想出力をシステム既定に、物理スピーカーをモニタリング出力に
- 入力ミックスの代わり: VoiceMeeter の仮想入力でマイク + システム音声を合成し、その仮想入力を koe の入力に指定

`system` / `both` モードは Windows でも画面収録 API で動くので、まずはアプリ内蔵モードを試すのが早い。
